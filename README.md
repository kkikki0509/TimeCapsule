## 주요 기능 및 구현
- 생성 기능  
  - 제목, 작성자, 내용, 비밀번호 입력값을 검증 후, fetch API의 POST 요청을 통해 JSON 데이터를 서버에 전송

- 조회 기능  
  - Spring Data JPA 쿼리 메서드를 활용하여 제목, 작성자, 날짜 기준으로 데이터를 조회하고, 반환된 JSON 데이터를 프론트엔드에서 목록에 표시

- 삭제 기능  
  - 사용자가 입력한 비밀번호를 검증한 후, 일치할 경우 해당 데이터 삭제

- 열람 제어 기능  
  - Letter 엔티티 내부에 isLocked() 메서드를 구현하여, 현재 날짜와 열람일(endDate)을 비교해 잠금 여부 판단
  - 잠금 상태에 따라 아이콘이 동적으로 변경되도록 처리
  - 서버에서 등록일(createdAt)과 1년 뒤의 열람일(endDate)을 자동 계산

- 유효성 및 입력값 검증  
  - 정규표현식을 활용하여 비밀번호는 숫자만 입력 가능하도록 제한
  - 필수 입력값이 비어 있을 경우 alert()를 통해 사용자에게 경고

## 프로젝트 진행 중 겪은 문제
- **문제 1**
  - 초기에는 전체 데이터를 서버에서 받아온 뒤 브라우저에서 필터링하는 방식으로 구현했으나, 데이터 양이 많아질 경우 성능 저하가 발생할 수 있다고 판단
- **해결**
  - 검색 조건에 따른 조회를 Spring에서 처리하도록 수정
---
- **문제 2**
  - 비밀번호 비교를 프론트엔드에서 처리하였으나, 보안에 취약하다는 점을 알게 됨
- **해결**
  - 백엔드에서 검증 후 삭제하도록 수정
---
- **문제 3**
  - Spring에서 date 타입에 빈 문자열이 전달될 경우 예외가 발생하는 문제를 확인
- **해결**
  - date 타입 입력값 유무에 따라 날짜(date) 값을 if-else문을 통해 전달하도록 로직을 수정

## 프로젝트 상태
- 기본적인 CRD 기능과 1년 기준 잠금 로직을 포함하여, 타임캡슐 서비스의 핵심 기능을 구현한 상태입니다.

- 추후 시간 여유가 있다면 메시지 수정 기능을 추가하고, 실제 서버 환경에 배포하여 다른 사용자도 이용할 수 있도록 생각하고 있습니다.
